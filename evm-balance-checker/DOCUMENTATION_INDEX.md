## üìù Documentation Standards

### File Naming Convention

- `README.md` - Main project documentation
- `QUICKSTART.md` - Quick start guides
- `*_GUIDE.md` - Comprehensive guides
- `*_REFERENCE.md` - Quick reference cards
- `*_OVERVIEW.md` - Architecture/overview docs
- `env.example` - Configuration templates
- `*.sh` - Executable scripts

### Documentation Structure

Each major document includes:
1. **Overview** - What is this?
2. **Quick Start** - Get running fast
3. **Detailed Guide** - Comprehensive instructions
4. **Examples** - Code examples
5. **Troubleshooting** - Common issues
6. **Reference** - Quick lookups
